Краткое руководство по процедуре сборки библиотеки DCMTK под ОС MS Windows (инструментарий MSVC 2013) 

Сборка библиотеки DCMTK представляет собой весьма нетривиальную задачу (если сранивать с другими известными библиотеками, например той же, эталлоной, на мой взгляд, в этом аспекте OpenCV).

Что же нам понадобится из инструментов? Всё как обычно:
 - клиент системы учёта версий git
 - утилита CMake
 - собственно, инструменты сборки MSVC 2013

 После подготовки инструментов приступаем к сборке DCMTK:
 
1) Итак, для начала, нам понадобится клонировать актуальную версию репозиотрия DCMTK:

   >> git clone https://github.com/commontk/DCMTK.git
   
2) Затем, запускаем CMake указываем, что исходники лежат в корне загруженного нами репозитория, указываем директорию в которую должна осуществялться сборка проекта.
   Оставив все оставшиеся опции умолчанию, нажимаем несколько раз "сonfigure" (не обращаем внимания на предупреждения - сам файл сообщает, что они предназначены для разработчиков библиотеки), затем просим утилиту сгенерировать проект для сборки библиотеки "generate"
   
3) Запускаем VisualStudio 2013 в режиме "Запуск от имени администратора" (это требуется только для установки библиотеки в каталог "C:\Program Files (x86)\DICOM" при сборке подпроекта INSTALL),
   собираем библиотеку и устанавливаем её
   
4) Вроде всё ок, для того, чтобы посмотреть как прилинковаться к библиотеке смотри файл DICOM.pri (там всё как обычно: указываем путь к заголовочным файлам и линкуемся с линковочным файлом).
   Так вот, вроде всё ок, но это не так (по крайней мере с версией библиотеки 3.6.1). При попытке собрать проект компилятор начинает ругаться на три функции из DCMTK. К счастью, эти функции оказываются не критичными (судя по именам они используются для ведения каких-то записей)
   и простое комментирование их тел решает проблему. Будем надеяться в будущем разработчики решат эту проблему по-человечески.
